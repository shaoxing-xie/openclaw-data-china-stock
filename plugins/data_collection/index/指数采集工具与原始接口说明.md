# 指数采集工具与原始接口说明

本文档说明各指数采集工具（含 `tool_fetch_index_opening`）如何调用原始数据接口，便于实现或对照其它指数工具的逻辑。

### 指数代码与数据源（与实现一致）

- **无白名单**：`tool_fetch_index_realtime` / `historical` / `minute` 对**任意可解析的 6 位指数代码**均尝试拉取；`index_code_utils.normalize_index_code_for_minute` 统一输入；`index_sina_symbol`：**39xxxx → `sz` + 代码，其余 → `sh` + 代码**。
- **ETF**：6 位以 **5** 或 **1** 开头在上述工具中会**自动改调 ETF 模块**，不走路径内「纯指数」逻辑。
- **历史日线**：入口**不强制**安装 akshare；优先缓存 → mootdx → Tushare 封装/直连 → 新浪 HTTP → AkShare → 东财（见 `fetch_historical.py`）。

## 一、工具与 data_type 对应关系

| 工具名 | data_type | 说明 |
|--------|-----------|------|
| tool_fetch_index_realtime | realtime | 指数实时行情 |
| tool_fetch_index_historical | historical | 指数历史日/周/月 K 线 |
| tool_fetch_cni_index_daily | cni_daily | 国证指数日频历史（AkShare `index_hist_cni`） |
| tool_fetch_csindex_index_daily | csindex_daily | 中证指数日频历史（AkShare `stock_zh_index_hist_csindex`） |
| tool_fetch_index_minute | minute | 指数分钟 K 线 |
| **tool_fetch_index_opening** | **opening** | **指数开盘数据（9:28 集合竞价）** |
| tool_fetch_global_index_spot | global_spot | 全球主要指数最新快照（延迟因数据源而异；非毫秒级实时） |

统一入口：`tool_fetch_index_data(data_type=..., ...)`，别名通过 `tool_runner` 注入 `data_type`。

## 二、tool_fetch_index_opening：调用的原始接口

开盘数据逻辑与 `fetch_realtime` 类似：都是“当前快照”，只是时间点不同（开盘前后取到的“今开”即集合竞价结果）。

### 原系统实现位置

- **`src/data_collector.py`**
  - `fetch_index_opening_data()`：当日开盘数据
  - `fetch_index_opening_history()`：历史开盘（用于均值等，内部用日线）

### 原始接口调用顺序（已实现：新浪主 → 东财备）

| 优先级 | 接口 | 说明 |
|--------|------|------|
| 1 | **akshare `stock_zh_index_spot_sina()`** | 新浪指数实时/集合竞价，**主用**。返回列含 代码/名称/今开/昨收/涨跌幅等。 |
| 2 | **akshare `stock_zh_index_spot_em(symbol=...)`** | 东方财富指数实时，**备用**。需轮询 symbol：`"沪深重要指数"`,`"上证系列指数"`,`"深证系列指数"`,`"中证系列指数"`，再按代码筛出目标指数。 |
| 3 | **mootdx `quotes(symbols)`** | 当 akshare 不可用或未取到结果时，用 mootdx 做开盘近似/快照。返回字段依赖 quotes 是否提供 `open/last_close` 等列。 |

### 字段映射（与 realtime 一致）

从 DataFrame 取列时兼容多种列名：

- **开盘价**：`今开` / `open` / `开盘` / `open_price`
- **昨收**：`昨收` / `close` / `close_yesterday` / `pre_close`
- **涨跌幅**：`涨跌幅` / `pct_chg` / `change_pct` / `涨跌%`
- **成交量**：`成交量` / `volume` / `vol` / `成交`
- **代码**：`代码` / `code` / `symbol`

### 默认指数列表（与原系统一致）

```python
index_codes = {
    "上证": "000001",
    "创业板": "399006",
    "深成指": "399001",
    "科创综指": "000688",
    "沪深300": "000300",
    "北证50": "899050"
}
```

实现时可按需改为 `index_codes: Optional[str] = None`（逗号分隔），再拆成列表/字典。

## 三、其它指数工具调用的原始接口（对照参考）

### 1. tool_fetch_index_realtime（fetch_realtime.py）

| 优先级 | 接口 | 说明 |
|--------|------|------|
| 1 | **mootdx** `quotes`（批量） | 主通道之一 |
| 2 | AkShare 全量快照：`stock_zh_index_spot_sina` / `stock_zh_index_spot_em`（短缓存） | 补缺 |
| 3 | **mootdx** 1 分钟 K 最后一根 | 兜底 |
| 降级 | `_get_fallback_data()` | 占位 |

### 2. tool_fetch_index_historical（fetch_historical.py）

**日线 daily**（周/月主要走 AkShare `index_zh_a_hist`）：

| 优先级 | 接口 | 说明 |
|--------|------|------|
| 1 | 本地缓存 `get_cached_index_daily`（全命中） | 若可用 |
| 2 | **mootdx** `bars(frequency=9)` | 日线 |
| 3 | `src.tushare_fallback.fetch_index_daily_tushare` | 若配置 Tushare |
| 4 | Tushare `pro.index_daily`（显式 token 时） | ts_code 由 `tushare_index_ts_code` |
| 5 | 新浪 HTTP `CN_MarketData.getKLineData`，`scale=240` | 直连 |
| 6 | `ak.index_zh_a_hist`（日/周/月） | 需 akshare |
| 7 | `ak.index_zh_a_hist_min_em(period="日k")` | 需 akshare |

### 3. tool_fetch_index_minute（fetch_minute.py）

| 优先级 | 接口 | 说明 |
|--------|------|------|
| 1 | 本地缓存 `get_cached_index_minute`（全命中） | 若可用 |
| 2 | **mootdx** 分钟 K | 与 period 对应 frequency |
| 3 | 新浪 HTTP `CN_MarketData.getKLineData`，`scale=1/5/15/30/60` | 主用 |
| 4 | `ak.index_zh_a_hist_min_em` | 可选，需 akshare |

## 四、实现 tool_fetch_index_opening 时的建议

1. **与 realtime 对齐**  
   开盘数据本质是“开盘时刻的 spot”，接口与 realtime 相同（东财 + 新浪），只是语义为“开盘数据”，返回里可强调 `opening_price`、`timestamp` 等。

2. **交易日与 mode**  
   与其它指数工具一致：先做交易日检查（`check_trading_day_before_operation`），支持 `mode="test"` 跳过检查。

3. **返回格式**  
   与《工具参考手册》约定一致，例如：
   - `success`, `message`, `data`
   - `data`: 列表，每项含 `index_code`/`code`, `name`, `opening_price`, `pre_close`, `change`/`change_pct`, `timestamp` 等。

4. **原始接口调用顺序**  
   采用实现中的顺序：先 **akshare `stock_zh_index_spot_sina()`**，取不到再 **akshare `stock_zh_index_spot_em(symbol=...)`**；若 akshare 不可用或两者均未取到，则退化为 **mootdx `quotes`** 返回快照近似，列名用上面“字段映射”做兼容。

5. **文件位置**  
   实现放在 `plugins/data_collection/index/fetch_opening.py`，对外提供 `fetch_index_opening()` 与 `tool_fetch_index_opening()`，由 `merged/fetch_index_data.py` 在 `data_type="opening"` 时导入并传入 `**kwargs`。

---

## 五、小结

- **tool_fetch_index_opening** 的数据源策略：akshare `stock_zh_index_spot_sina()` → 取不到再 akshare `stock_zh_index_spot_em(symbol=...)`；若 akshare 不可用或无结果，则退化为 mootdx `quotes` 返回快照近似。
- 当退化到 mootdx `quotes` 时：`opening_price` 为近似值（取决于 quotes 是否返回 `open/last_close` 等字段）。
- 区别只在于业务含义（开盘/集合竞价）和返回字段命名（如 `opening_price`）。
- 当 akshare 不可用时，会退化为 **mootdx quotes** 返回的快照字段（opening 为近似），以保证工作流不因依赖缺失而完全中断。
